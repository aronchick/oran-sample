Real-time telemetry collection for OpenShift Single Node deployments. Expanso captures PTP sync, SR-IOV metrics, and FEC accelerator data—delivered via OTLP to your observability stack.

Quick Start Guide View Architecture

Sub-100ns PTP Sync

O-RAN DUs require nanosecond-precision timing. We monitor ptp4l and phc2sys offsets in real-time, flagging violations instantly.

Edge-Native Processing

Expanso runs directly on SNO nodes with minimal footprint. Bloblang transformations normalize telemetry before it leaves the edge.

Standard OTLP Output

Metrics flow via OpenTelemetry Protocol to any backend—Grafana, Prometheus, Red Hat Observability, or your existing stack.

Deploy in Minutes

One command to deploy on OpenShift Single Node

1 Quick Start (Local) 

# Install Expanso Edge
curl -sL https://get.expanso.io/edge/install.sh | bash

# Run with OTLP output (adjust endpoint)
OTLP_ENDPOINT=http://localhost:4318 \
  expanso-edge run pipeline-otlp.yaml

2 OpenShift SNO Deploy 

# Clone the repo
git clone https://github.com/expanso-io/oran-demo
cd oran-demo

# Deploy to OpenShift
oc apply -k deploy/openshift/

# Customize OTLP endpoint
oc set env deployment/expanso-oran-collector \
  OTLP_ENDPOINT=https://your-backend:4318

Configuration Options

Environment Variable Default Description
OTLP_ENDPOINT http://localhost:4318 OTLP HTTP receiver endpoint
EMIT_INTERVAL 5s Telemetry emission interval
NODE_NAME du-sno-worker DU node identifier
CLUSTER_NAME sno-edge-01 OpenShift cluster name
