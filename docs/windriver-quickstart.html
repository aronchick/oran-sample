Expanso Edge Deployment Guide

Deploy Expanso data pipelines on Wind River Studio-managed Kubernetes clusters for vRAN and O-RAN telemetry processing.

Overview

This guide shows how to deploy Expanso Edge alongside Wind River Studio to add real-time data pipeline processing and edge AI capabilities to your existing infrastructure.

What Expanso Adds to Studio

  • Real-time data transformation - Bloblang processors normalize and enrich telemetry
  • Edge AI inference - Run ML models directly on edge nodes
  • OTLP output - Standard OpenTelemetry Protocol for any observability backend
  • Cloud-managed pipelines - Update configurations across thousands of nodes

Prerequisites

  • Wind River Studio - Cloud Platform with managed edge clusters
  • kubectl access - Configured to access your Studio-managed cluster
  • Expanso Cloud account - Register at cloud.expanso.io

Verify Cluster Access

# Verify kubectl is configured for your Studio cluster
kubectl cluster-info
kubectl get nodes

# Ensure you have a namespace for Expanso
kubectl create namespace expanso-system

Deployment Steps

1 Register with Expanso Cloud

Create your Expanso Cloud account and generate credentials for the edge nodes.

  1. Go to cloud.expanso.io
  2. Create a new Network (e.g., "wind-river-prod")
  3. Generate a bootstrap token

Note: The bootstrap token is a one-time credential. Store it securely.

2 Create Bootstrap Secret

Store the bootstrap token as a Kubernetes secret:

kubectl create secret generic expanso-bootstrap \
  -n expanso-system \
  --from-literal=token=<BOOTSTRAP_TOKEN>

Verify the secret:

kubectl get secret expanso-bootstrap -n expanso-system

3 Deploy Expanso Edge

Deploy the Expanso Edge agent to your Studio-managed cluster:

kubectl apply -f expanso-edge-deployment.yaml -n expanso-system

Or use the Helm chart for more configuration options:

helm repo add expanso https://charts.expanso.io
helm install expanso-edge expanso/expanso-edge \
  -n expanso-system \
  --set bootstrap.secretName=expanso-bootstrap

The agent will:

  1. Read the bootstrap token from the secret
  2. Register itself with Expanso Cloud
  3. Begin polling for pipeline configurations

Verify registration:

kubectl logs -f deployment/expanso-edge -n expanso-system

4 Deploy Pipeline via Expanso Cloud

Create and deploy a pipeline through Expanso Cloud:

  1. Go to cloud.expanso.io
  2. Navigate to your Network
  3. Click Create Pipeline
  4. Paste your pipeline configuration (example below)
  5. Click Deploy

The pipeline will be pushed to all nodes in the network.

5 Configure Output to Studio Analytics

Point the pipeline output to your Studio Analytics OTLP endpoint:

output:
  http_client:
    url: http://<studio-analytics-endpoint>:4318/v1/metrics
    headers:
      Content-Type: application/json
    verb: POST
    retries: 3
    timeout: 5s

Or configure multiple outputs for local + central observability:

output:
  broker:
    outputs:
      - http_client:  # Local Studio Analytics
          url: http://studio-analytics-local:4318/v1/metrics
      - http_client:  # Central aggregation
          url: http://studio-analytics-central:4318/v1/metrics

Example: vRAN Telemetry Pipeline

This pipeline collects O-RAN metrics and transforms them to OTLP format:

input:
  generate:
    interval: 5s
    mapping: |
      root.timestamp = now().ts_unix()
      root.du_id = "du-001"
      root.ptp4l_offset_ns = random_int(max: 200) - 50
      root.cpu_pct = random_int(min: 15, max: 65)
      root.prb_dl_pct = (random_int(min: 200, max: 800) / 10.0)

pipeline:
  processors:
    - mapping: |
        let ts = (this.timestamp * 1000000000).string()

        root.resourceMetrics = [{
          "resource": {
            "attributes": [
              {"key": "service.name", "value": {"stringValue": "vran-du"}},
              {"key": "du.id", "value": {"stringValue": this.du_id}}
            ]
          },
          "scopeMetrics": [{
            "scope": {"name": "du-telemetry", "version": "1.0.0"},
            "metrics": [
              {
                "name": "du.ptp4l_offset_ns",
                "unit": "ns",
                "gauge": {
                  "dataPoints": [{"timeUnixNano": $ts, "asInt": this.ptp4l_offset_ns}]
                }
              },
              {
                "name": "du.cpu_pct",
                "unit": "%",
                "gauge": {
                  "dataPoints": [{"timeUnixNano": $ts, "asInt": this.cpu_pct}]
                }
              }
            ]
          }]
        }]

output:
  http_client:
    url: http://studio-analytics:4318/v1/metrics
    headers:
      Content-Type: application/json
    verb: POST

O-RAN Telemetry Metrics

Key metrics for vRAN/O-RAN deployments:

Metric Unit Description
du.ptp4l_offset_ns ns PTP clock offset from grandmaster
du.phc2sys_offset_ns ns System clock to PTP hardware clock offset
du.cpu_pct % CPU utilization on isolated cores
du.prb_dl_pct % Physical Resource Block utilization (downlink)

PTP Compliance Thresholds

Per O-RAN specifications:

Metric Threshold Status
ptp4l offset ±100ns DEGRADED if exceeded
phc2sys offset ±50ns DEGRADED if exceeded
Lock state != LOCKED CRITICAL

Troubleshooting

Expanso Edge not registering

# Check pod status
kubectl get pods -n expanso-system

# Check logs for registration errors
kubectl logs deployment/expanso-edge -n expanso-system

# Verify secret exists
kubectl get secret expanso-bootstrap -n expanso-system

Pipeline not processing data

# Check pipeline status in Expanso Cloud
# Go to cloud.expanso.io > Network > Pipelines

# Check Expanso Edge logs for pipeline errors
kubectl logs deployment/expanso-edge -n expanso-system | grep -i error

# Verify network connectivity to output endpoint
kubectl exec -it deployment/expanso-edge -n expanso-system -- \
  curl -v http://studio-analytics:4318/v1/metrics

Metrics not appearing in Studio Analytics

# Verify OTLP endpoint is reachable
kubectl exec -it deployment/expanso-edge -n expanso-system -- \
  wget -q -O - http://studio-analytics:4318/health

# Check output configuration in pipeline
# Ensure Content-Type header is set to application/json
